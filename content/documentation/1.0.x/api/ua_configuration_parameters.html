Mandatory parameters h4(#parameter_uri). @uri@ SIP URI associated to the User Agent (@String@). This is a SIP address given to you by your provider. CODE_BEGIN "javascript" uri: "sip:alice@example.com" CODE_END h4(#parameter_ws_servers). @ws_servers@ Set of WebSocket URIs to connect to. If not specified ports 80 and 443 are taken for non secure WebSocket connections and connections over SSL/TLS, respectively. This parameter can be expressed in multiple ways: * Single @String@ to define a single WebSocket URI. * @Array@ of @Strings@ to define multiple WebSocket URIs. * @Array@ of @Objects@ to define multiple WebSocket URIs with weight. URIs with higher weight value are used prior to those with lower value. CODE_BEGIN "javascript" ws_servers: "ws://sip-ws.example.com" CODE_END CODE_BEGIN "javascript" ws_servers: "wss://sip-ws.example.com:8443/sip?KEY=1234" CODE_END CODE_BEGIN "javascript" ws_servers: [ "ws://sip-ws-1.example.com", "ws://sip-ws-2.example.com", ] CODE_END CODE_BEGIN "javascript" ws_servers: [ {ws_uri: "ws://sip-ws-1.example.com", weight: 10}, {ws_uri: "ws://sip-ws-2.example.com", weight: 1} ] CODE_END h4(#parameter_sockets). @sockets@ Set of JsSIP.Socket instances. This parameter can be expressed in multiple ways: * Single @JsSIP.Socket@ instance. * @Array@ of @JsSIP.Socket@ instances. * @Array@ of @Objects@ defining a @JsSIP.Socket@ instance with weight. Sockets with higher weight value are used prior to those with lower value. CODE_BEGIN "javascript" sockets: socket CODE_END CODE_BEGIN "javascript" sockets: [ socket1, socket2, ] CODE_END CODE_BEGIN "javascript" sockets: [ {socket: socket1, weight: 10}, {socket: socket2, weight: 1} ] CODE_END h2(#section_optional). Optional parameters h4(#parameter_authorization_user). @authorization_user@ Username (@String@) to use when generating authentication credentials. If not defined the value in @uri@ parameter is used. CODE_BEGIN "javascript" authorization_user: "alice123" CODE_END h4(#parameter_connection_recovery_max_interval). @connection_recovery_max_interval@ Maximum interval (@Number@) in seconds between WebSocket reconnection attemps. Default value is @30@. CODE_BEGIN "javascript" connection_recovery_max_interval: 60 CODE_END h4(#parameter_connection_recovery_min_interval). @connection_recovery_min_interval@ Minimum interval (@Number@) in seconds between WebSocket reconnection attempts. Default value is @2@. CODE_BEGIN "javascript" connection_recovery_min_interval: 4 CODE_END h4(#parameter_display_name). @display_name@ Descriptive name (@String@) to be shown to the called party when calling or sending IM messages. It must NOT be enclosed between double quotes even if the given name contains multi-byte symbols (JsSIP will always enclose the @display_name@ value between double quotes). CODE_BEGIN "javascript" display_name: "Alice ¶€ĸøĸø" CODE_END h4(#parameter_hack_ip_in_contact). @hack_ip_in_contact@ Set a random IP address as the host value in the Contact header field and Via sent-by parameter. Useful for SIP registrars not allowing domain names in the Contact URI. Valid values are @true@ and @false@ (@Boolean@). Default value is a @false@. CODE_BEGIN "javascript" hack_ip_in_contact: true CODE_END h4(#instance_id). @instance_id@ @String@ indicating the UUID URI to be used as instance ID to identify the UA instance when using GRUU. CODE_BEGIN "javascript" instance_id: "uuid:8f1fa16a-1165-4a96-8341-785b1ef24f12" CODE_END CODE_BEGIN "javascript" instance_id: "8f1fa16a-1165-4a96-8341-785b1ef24f12" CODE_END h4(#parameter_no_answer_timeout). @no_answer_timeout@ Time (in seconds) (@Integer@) after which an incoming call is rejected if not answered. Default value is @60@. CODE_BEGIN "javascript" no_answer_timeout: 120 CODE_END h4(#parameter_session_timers). @session_timers@ Enable Session Timers (as per "RFC 4028":https://tools.ietf.org/html/rfc4028). Default value is @true@. CODE_BEGIN "javascript" session_timers: false CODE_END h4(#parameter_password). @password@ SIP Authentication password (@String@). The plain SIP password. JsSIP deletes this value from its internal memory after the first successful authentication and, instead, stores the resulting @ha1@ and @realm@. CODE_BEGIN "javascript" password: "1234" CODE_END h4(#parameter_realm). @realm@ SIP Authentication realm (@String@). Just useful if plain SIP @password@ is not given, so it also requires @ha1@ to be provided. CODE_BEGIN "javascript" realm: "mydomain.com" CODE_END NOTE for Asterisk users: Asterisk always uses "asterisk" as @realm@. h4(#parameter_ha1). @ha1@ SIP Authentication HA1 hash (@String@) for the Digest authentication. If given (and plain SIP @password@ is not given) the parameter @realm@ is also required. Otherwise authentication is not feasible. CODE_BEGIN "javascript" ha1: "270fe19ce3890bd85e105998b0a75cf0" CODE_END h4(#parameter_register). @register@ Indicate if JsSIP User Agent should register automatically when starting. Valid values are @true@ and @false@ @(Boolean)@. Default value is @true@. CODE_BEGIN "javascript" register: false CODE_END h4(#parameter_register_expires). @register_expires@ Registration expiry time (in seconds) @(Integer)@. Default value is @600@. CODE_BEGIN "javascript" register_expires: 300 CODE_END h4(#parameter_registrar_server). @registrar_server@ Set the SIP registrar URI. Valid value is a SIP URI without username. Default value is @null@ which means that the registrar URI is taken from the @uri@ parameter (by removing the username). CODE_BEGIN "javascript" registrar_server: 'sip:registrar.mydomain.com' CODE_END h4(#parameter_use_preloaded_route). @use_preloaded_route@ If set to @true@ every SIP initial request sent by JsSIP includes a Route header with the SIP URI associated to the WebSocket server as value.
